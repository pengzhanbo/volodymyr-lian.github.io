gulp学习（一）—— API 说明

  • gulp
  • node
  • 笔记

gulp 是一款前端自动化构建工具，是自动化构建项目的利器，不仅能对网站资源进行优化，而且能够避免我们在开发过程中进行多次的重复任务，gulp 自动帮你完成！
使用gulp 能大大的提高我们的开发效率！
gulp 是基于 Nodejs的自动化构建工具，一般用于处理包括html/css/javascript/jade/less/sass/stylus/typescript/coffeescript/ES6/images 等资源文件的测试、检查、合并、压缩、格式化、浏览器自动刷新，部署文件生成，监听文件变动 等一系列任务。
本文将针对 gulp 的API进行简单说明。

1. gulp.src(globs[,option])

src 方法指定的需要处理的源文件的路径，gulp.src 返回的当前文件流可通过管道 pipe 往下一级传入。
@param globs
_需要处理的源文件匹配符路径。类型String,Array必填
src/a.js : 指定具体的文件
* : 匹配所有的文件 例: src/*.html 匹配src 路径下所有html文件(不包含子文件夹内的html文件)
**: 匹配0个或多个子文件夹，包括当前层级的文件夹 例: src/**/*.js 匹配src路径下包括所有子文件夹中的所有 js文件
{}: 匹配多个属性 例: src/{a,b}.js 匹配src路径下的a.js和b.js
! : 排除文件 例: !src/a.js 不包含 src/a.js 文件
@param option 可选。包含3个属性值buffer、read、base_
option.buffer:类型Boolean 默认:true 设置为false,将返回file.content的流并且不缓冲文件，处理大文件时非常有用；
option.read:类型Boolean 默认:true 设置false，将不执行读取文件操作，返回null；
option.base:类型String 设置输出路径以某个路径的某个组成部分为基础向后拼接。

2. gulp.dest(paths[,options])

dest 方法指定的处理后的文件的输出路径
@param paths
指定文件输出路径，也可以定义函数返回输出路径 必填
@param options
可选。包含2个属性 cwd、mode
options.cwd : 类型String 默认：process.cwd()：前脚本的工作目录的路径 当文件输出路径为相对路径将会用到
option.mode : 类型String 0777 指定被创建文件夹的权限；

3. gulp.task(name[,deps],callback)

task 定义一个gulp任务
@param name
指定任务的名称。类型：String
@param deps
指定任务的依赖的任务（被依赖的任务应该返回文件流）类型：String/Array 可选

@param callback
该任务调用的文件处理操作 必选

4. gulp.watch(glob [, opts], tasks) 或 gulp.watch(glob [, opts, callback__])

watch 方法用于监听文件变化，文件修改则触发执行相应的任务
@param glob
需要处理的源文件匹配符路径 必填 String|Array
@param opts
可选 Object https://github.com/shama/gaze
@param tasks
需要执行的任务的名称数组 必填 String|Array
@param callback
每个文件变化执行的回调函数 选填 Function
